Issues utilizados en la segunda clase del curso
¿Qué son los Issues y para qué sirven?

Los Issues de GitHub permiten gestionar y dar seguimiento al desarrollo de un proyecto sin necesidad de utilizar herramientas externas. Su objetivo principal es facilitar la organización, comunicación y planificación dentro de un equipo de trabajo.

Principales usos de los Issues
🐞 Reporte de errores (Bug Tracking)

Cuando se detecta un problema en el código, se puede crear un Issue para documentarlo. Generalmente se describe el error, los pasos para reproducirlo y el comportamiento esperado.

✨ Propuesta de nuevas funcionalidades

Los Issues también sirven para sugerir mejoras, nuevas características, cambios en la interfaz o modificaciones en la lógica del proyecto.

📋 Gestión de tareas

Permiten dividir proyectos complejos en actividades más pequeñas y asignarlas a distintos integrantes del equipo, facilitando el seguimiento del trabajo.

💬 Espacio para discusión

Pueden utilizarse para plantear preguntas o debatir aspectos relacionados con el diseño, la arquitectura o el funcionamiento de una aplicación.

Elementos que puede contener un Issue

Los Issues ofrecen varias herramientas que ayudan a organizar mejor el trabajo:

👤 Responsables (Assignees)

Permiten asignar una tarea a una o varias personas para identificar quién se encargará de resolverla.

🏷️ Etiquetas (Labels)

Se utilizan para clasificar los Issues según su naturaleza, por ejemplo: bug, enhancement, documentation o help wanted.

🎯 Hitos (Milestones)

Sirven para agrupar varios Issues dentro de un objetivo o versión específica del proyecto, como por ejemplo una entrega denominada "Versión 1.0".

🔗 Relación con el código

Un Issue puede vincularse a un Pull Request. De esta manera, cuando los cambios son aprobados e integrados al repositorio, el Issue asociado puede cerrarse automáticamente.

GitHub Projects y el tablero Kanban

GitHub ofrece una herramienta llamada GitHub Projects, que permite visualizar y administrar los Issues mediante un tablero Kanban. Esta vista facilita el seguimiento del progreso de un proyecto de forma gráfica e intuitiva.

Puede compararse con una pizarra llena de notas adhesivas organizadas en columnas, donde cada nota representa un Issue y cada columna corresponde a una etapa del flujo de trabajo.

Funcionamiento básico de un tablero Kanban

Aunque las columnas pueden personalizarse según las necesidades del equipo, normalmente se utilizan tres estados principales:

📌 Todo (Por hacer)

Contiene todas las tareas pendientes, ideas, mejoras o errores reportados que todavía no han comenzado a desarrollarse.

🚧 In Progress (En progreso)

Cuando una persona empieza a trabajar en una tarea, el Issue se mueve a esta columna. Esto permite que todo el equipo conozca qué actividades están siendo desarrolladas en ese momento.

✅ Done (Hecho)

Incluye las tareas finalizadas. Una vez que el trabajo ha sido completado, revisado e integrado al proyecto, el Issue se traslada a esta columna y se marca como cerrado.
